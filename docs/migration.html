Migração de 2.x para 3.0

Bootstrap 3 não é compatível com as versões anteriors v2.x. Utilize esta seção como um guia geral para atualizar de v2.x para v3.x. Para uma visão mais abrangente, veja o que há de novo? no anúncio de liberação da versão v3.0.

O que foi removido?

Os seguintes elementos foram removidos ou alterados na v3.0.

Elemento Removido da 2.x Equivalente na 3.0
Ações de formulário .form-actions N/A

Campos com tamanho de largura fixa .input-mini .input-small .input-medium .input-large .input-xlarge .input-xxlarge Use .form-control e o sistema de grid como substituto.
Bloco de nível de campo de formulário .input-block-level Não possui um equivalente direto, mas controles de formulário são similares.

Notas adicionais

Outras alterações na v3.0 não são imediatamente aparentes. Classes base, estilos chave e comportamentos tem sido ajustados para flexibilidade e nossa abordagem mobile first. Aqui está uma lista parcial:

  • Por padrão, controles de formulário baseados em texto agora recebem uma estilização mínima. Para focar cores e arredondar cantos, aplique a classe .form-control no elemento para estilizar.
  • Controles de formulário baseados em texto com a classe .form-control aplicado agora possuem largura 100% por padrão. Encapsule campos dentro de <div class="col-*"></div> para controlar a largura dos campos.
  • .badge não mais possui classes contextuais (-success,-primary,etc..).
  • .btn devem usar também .btn-default para serem o botão "default".
  • .row agora são fluídas.
  • Imagens não são responsívas por padrão. Use .img-responsive para <img> com tamanhos fluídos.
  • Os ícones, agora .glyphicon, agora são baseados em fonte. Ícones também requerem uma base e uma classe icon (e.g. .glyphicon .glyphicon-asterisk).
  • Typehead foi removido, para o uso do Twitter Typeahead.
  • Marcações de modal mudaram significativamente. As seções .modal-header, .modal-body, e .modal-footer agora estão encapsuladas em .modal-content e .modal-dialog para uma melhor estilização e comportamento em dispositívos móveis. Você também não deveria mais aplicar .hide para .modal em suas marcações.
  • As of v3.1.0, o HTML carregado pelos opções de modal remote está agora injetado dentro de .modal-content (de v3.0.0 para v3.0.3, dentro de .modal) ao invés de dentro de .modal-body. Iss permite que você varie facilmente o cabeçalho e o rodapé do modal também, não somente o corpo do modal.
  • Eventos Javascript estão em um namespace. Por exemplo, para manipular o evento "mostrar" de um modal, use 'show.bs.modal'. Para "mostrar" abas use 'shown.bs.tab', etc.

Para mais informações sobre atualização para v3.0, e trechos de código da comunidade, veja Bootply.
