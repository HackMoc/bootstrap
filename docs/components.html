Como usar

Por razões de desempenho, todos os ícones requerem uma classe base e uma classe específica para cada ícone. Para usar, coloque o código abaixo em qualquer lugar. Certifique-se de deixar um espaço entre o ícone e o texto, para um espaçamento conciso.

Não misture com outros componentes

Classes de ícones não podem ser diretamente combinadas com outros componentes. Não deve ser declaradas com outras classes no mesmo elemento. No lugar, adicione <span> aninhado e aplique a classe do icone ao <span>.

Alinhamento

Por padrão, o menu dropdown é automaticamente posicionado 100% a partir do topo juntamente a esquerda de seu antecessor. Adicione .dropdown-menu-right a .dropdown-menu para alinhar a direita o menu dropdown.

Posicionamento adicional pode ser requerido

Menus dropdown são posicionados automaticamente via CSS com o fluxo normal do documento. Isso significa que os menus dropdowns podem ser cortados pelos seus antecessores através da propriedade overflow ou exibidas fora dos limites. Cuide destes ajustes na medida que forem aparecendo.

Depreciado .pull-right Alinhamento

Desde a versão v3.1.0, nós depreciamos os menus com .pull-right. Para alinhar um menu a direita, use .dropdown-menu-right. Componentes nav alinhados a direita na navbar use uma mistura das versões dessa classe para alinhar. Para sobrescreve-la, use .dropdown-menu-left.

Manipulando bordas

Devido as especifidades do HTML e do CSS usadas para justificar botões (isto é display: table-cell), as bordas entre eles são dobradas. Em um agrupamento regular, margin-left: -1px é usado para empilhar as bordas, invés de removê-las. Entretanto, margin não funciona com display: table-cell. Como resultado, dependendo da sua customização do Bootstrap, você pode desejar remover ou recolorir as bordas.

Barra de navegação

Barra de navegação padrão

Barras de navegação são meta componentes responsívos que servem como cabeçalhos de navegação para sua aplicação ou site. Eles iniciam escondidos (e são podem ser alternados) em visualizações em dispositivos móveis e se tornam horizontais de acordo com o aumento da largura do espaço de de visualização disponível.

Sobrecarregando conteúdo

Como Bootstrap não sabe a quantidade de espaço que o conteúdo em sua barra de navegação vai precisar, você poderá encontrar alguns problemas com o encapsulamento de conteúdo em uma segunda linha. Para resolver isso, você pode:

  1. Reduzir a quantidade ou largura dos item da barra de navegação.
  2. Esconder certos itens do barra de navegação em certos tamanhos de tela utilizando classes utilitárias responsivas.
  3. Altere o momento em que sua barra de navegação alterna entre escondida e modo horizontal. Customize a variável @grid-float-breakpoint ou adicione sua própria media query.

Requer JavaScript

Se o Javascript está desabilitado e a área de visualização é limitada o bastante para que a barra de navegação se esconda, será impossível expandir a barra de navegação e visualizar seu conteúdo com .navbar-collapse.

Alertas

Fornecer mensagens de feedback contextuais para ações típicas de usuários com alertas de mensagens disponibilidades e flexíveis. Para ícone de fechar em linha, use o plugin de alertas jQuery.

Exemplos

Encapsule qualquer texto e um botão de fechar opcional em .alert e uma das quatro classes contextuais (e.x., .alert-success) para mensagens de alerta básicas.

Classes não padrão

Alertar não possuem classes padrão, somente classes base e modificadoras. O alerta cinza padrão não faz muito sentido, então se você está convidado a especificar um tipo de classe contextual. Escolha entre sucess, info, warning ou danger.

Muito bem! Você leu essa mensagem de alerta com sucesso.
Atenção! Esta mensagem precisa da sua atenção, mas não é super importante.
Alerta! É melhor você olhar no espelho, você não está com uma aparência muito boa.
Erro! Mude algumas coisas e tente submeter novamente.

Progress bars

Provide up-to-date feedback on the progress of a workflow or action with simple yet flexible progress bars.

Cross-browser compatibility

Progress bars use CSS3 transitions and animations to achieve some of their effects. These features are not supported in Internet Explorer 9 and below or older versions of Firefox. Opera 12 does not support animations.

Default media

The default media allow to float a media object (images, video, audio) to the left or right of a content block.
